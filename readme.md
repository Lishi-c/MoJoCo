## 摘要

本研究立足于 Google DeepMind 推出的 MuJoCo 高性能物理引擎及其模型预测控制（MPC）框架，旨在构建一套集成实时仿真与高级可视化技术的汽车仪表系统。本项目不仅深入挖掘了 MJCF 模型的层次结构与 C++ API 数据接口，更在图形表现上突破了传统的 2D 叠加模式。通过应用 **OpenGL 混合渲染（Blending）** 与 **三维广告牌（Billboard）投影算法**，实现了一个具有**磨砂玻璃质感、黑色高对比度字符的扇形悬浮仪表盘**。该系统能够精准提取车辆的线速度与位置向量，并在 3D 仿真环境中进行低延迟映射。同时，本项目通过随机目标驱动的 MPC 闭环控制逻辑，验证了系统在动态轨迹追踪下的稳健性。

## 2. 增强型渲染方案设计

本项目在可视化层面采用了非对称的**扇形 HUD（抬头显示）**风格设计，以下是具体的技术实现细节。

### 2.1 扇形玻璃底座（Glass-Style Arc Base）

不同于常规的 `GL_TRIANGLE_FAN` 绘制圆，本项目设计了 `drawArcBackground` 逻辑。利用三角函数迭代生成  的顶点序列，并结合 `glBlendFunc(GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA)` 实现了具有深蓝色调的半透明玻璃质感。表盘外缘增加了一层青色发光边缘（Cyan Glow），提升了科技感。

### 2.2 黑色高对比度字符系统

为了提升在复杂背景下的可读性，本项目将所有仪表盘文字、数字及单位（KM/H）强制渲染为**高饱和度纯黑色**。

* **数码化渲染**: 采用自定义的七段数码管逻辑，通过 `drawDigit` 函数以线段组合模拟数字。
* **单位布局**: 重新设计了 `drawUnitLabel` 字符比例，使其嵌套在扇形底部中心位置。

### 2.3 动态激光指针（Laser Needle）

指针放弃了传统的三角块结构，改用**渐变激光线段**。

* **起始点优化**: 线段从扇形圆心出发，指向当前的刻度位置。
* **视觉反馈**: 在指针末端叠加了一个 `GL_POINTS` 高亮光点，通过光点的位移直观反映加速度变化。

## 4. 技术挑战与补救方案

| 技术难点 | 产生原因 | 最终解决方案 |
| --- | --- | --- |
| **Z-fighting 闪烁** | 仪表盘底色与刻度线处于同一 Z 平面 | 在绘制不同层级间插入微小的 `glTranslatef(0,0,0.1f)` 物理间隔 |
| **字符颜色遮盖** | 原先白色字符在浅色地面上难以识别 | 重写颜色常量，使用 `glColor4f(0, 0, 0, 1.0f)` 实现纯黑文字渲染 |
| **UI 旋转失控** | 车辆转弯时 UI 随之翻转 | 引入 ModelView 矩阵重置算法，剥离模型空间旋转分量 |
| **MPC 追踪失败** | 目标点过远导致控制量饱和 | 限制随机目标点的坐标范围在  区域内 |
