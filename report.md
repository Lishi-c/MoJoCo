## 摘要

本研究立足于 Google DeepMind 推出的 MuJoCo 高性能物理引擎及其模型预测控制（MPC）框架，旨在构建一套集成实时仿真与高级可视化技术的汽车仪表系统。本项目不仅深入挖掘了 MJCF 模型的层次结构与 C++ API 数据接口，更在图形表现上突破了传统的 2D 叠加模式。通过应用 **OpenGL 混合渲染（Blending）** 与 **三维广告牌（Billboard）投影算法**，实现了一个具有**磨砂玻璃质感、黑色高对比度字符的扇形悬浮仪表盘**。该系统能够精准提取车辆的线速度与位置向量，并在 3D 仿真环境中进行低延迟映射。同时，本项目通过随机目标驱动的 MPC 闭环控制逻辑，验证了系统在动态轨迹追踪下的稳健性。

## **二、技术方案**

### **2.1 系统架构**

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   MJCF场景文件   │───▶│  MuJoCo模型加载  │───▶│   物理仿真循环   │
└─────────────────┘    └─────────────────┘    └────────┬────────┘
                                                       │
┌─────────────────┐    ┌─────────────────┐    ┌────────▼────────┐
│   仪表盘渲染     │◀───│  数据提取与转换  │◀───│   mjData状态    │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```
### **2.2 数据流程**
 **仿真数据 (mjData)** 

   ↓

 **提取模块 (DashboardDataExtractor)** 

   ↓

 **结构化数据 (DashboardData)** 

   ↓

 **渲染模块 (DashboardRenderer)** 

   ↓

 **屏幕显示 (OpenGL 2D HUD)** 

### **2.3 渲染方案**
采用**OpenGL立即模式**实现2D覆盖层渲染，优点包括：
- 实现简单，适合初学者
- 无需复杂着色器与纹理管理
- 便于快速调试与迭代

**渲染流程：**
1. 保存当前OpenGL状态（矩阵、属性）
2. 切换到2D正交投影
3. 禁用深度测试与光照
4. 绘制仪表盘各组件
5. 恢复原始状态

## **三、实现细节**

### **3.1 场景创建**
- **文件：** `scenes/car_simple.xml`
- **结构：** 包含车身、四个车轮、自由关节、执行器与传感器
- **特点：** 使用`<freejoint/>`实现车辆的自由移动，车轮通过`<motor>`执行器控制


### **3.2 数据获取**
**关键代码：**
```cpp
void DashboardDataExtractor::update(const mjData* d, DashboardData& dash) {
    // 速度计算
    dash.speed = sqrt(d->qvel[0] * d->qvel[0] + d->qvel[1] * d->qvel[1]);
    dash.speed_kmh = dash.speed * 3.6;
    
    // 转速模拟（与速度成正比）
    dash.rpm = dash.speed_kmh * 40;
    if (dash.rpm > 8000) dash.rpm = 8000;
    
    // 位置获取
    dash.position_x = d->qpos[0];
    dash.position_y = d->qpos[1];
    dash.position_z = d->qpos[2];
}

```
### 3.3 仪表盘渲染

#### 3.3.1 速度表
- **表盘**：灰色圆形背景，白色刻度线（每20 km/h一个刻度）。
- **指针**：红色线条，根据速度值计算角度（0 km/h对应225°，200 km/h对应-45°）。
- **中心点**：小型黑色圆点。

**关键代码**：
```cpp
float angle = M_PI * 0.75 - (speed_kmh / 200.0) * M_PI * 1.5;
glBegin(GL_LINES);
glVertex2f(center_x, center_y);
glVertex2f(center_x + radius * cos(angle), 
           center_y + radius * sin(angle));
glEnd();
```

#### 3.3.2 转速表
- **表盘**：深蓝色圆形背景，每1000 RPM一个刻度。
- **红色区域**：6000-8000 RPM为“红区”，用半透明红色圆弧表示。
- **指针**：绿色线条，角度随RPM线性变化。

#### 3.3.3 数字面板
- **油量条**：绿色进度条，随仿真时间缓慢减少（模拟油耗）。
- **温度条**：蓝色到红色渐变，随转速升高而增加。
- **位置显示**：实时输出X、Y坐标（控制台）。

##  **四、技术挑战与补救方案** 

| 技术难点 | 产生原因 | 最终解决方案 |
| --- | --- | --- |
| **Z-fighting 闪烁** | 仪表盘底色与刻度线处于同一 Z 平面 | 在绘制不同层级间插入微小的 `glTranslatef(0,0,0.1f)` 物理间隔 |
| **字符颜色遮盖** | 原先白色字符在浅色地面上难以识别 | 重写颜色常量，使用 `glColor4f(0, 0, 0, 1.0f)` 实现纯黑文字渲染 |
| **UI 旋转失控** | 车辆转弯时 UI 随之翻转 | 引入 ModelView 矩阵重置算法，剥离模型空间旋转分量 |
| **MPC 追踪失败** | 目标点过远导致控制量饱和 | 限制随机目标点的坐标范围在  区域内 |
